Getting Started

Our-Lovely-CLI (olc) needs to be initialized in order to customize your project setup.

★ ★ ★

Initialize

Start using our-lovely-cli by initializing it inside an existing git repository:

git olc-rerun-setup

You'll have to answer a few questions and generate some tokens.

Features

  • Develop new features for upcoming releases
  • May exist on github repo, or may exist in developers repos only

★ ★ ★

Start a new feature

Development of new features starting from the 'develop' branch.

Start developing a new feature with

git workon TEAM-1234

This action performs the following:

  • Create branch 'feature/TEAM-1234'
  • Switch to the new branch
  • Assign Jira issue to you
  • Transition Jira issue to 'In Progress' status

Make a commit

Make a commit with an interactively formatted commit message
This action performs the following:

  • Select a gitmoji
  • Select a JIRA issue
  • Add a one-line summary
  • Add a multi-line description
  • Add a multi-line test plan
git cm -s -S -a

Start a pull request

Create a pull request for review on GitHub and "start" the Jira issue!
This will transition the Jira issue to "In Review" status.

git pull-request

Land a feature

Land a pull-request on GitHub and transition Jira issue to 'Landed'
This action performs the following:

  • Merges TEAM-1234 into 'develop'
  • Removes the feature branch
  • Switches back to 'develop' branch
  • Transition Jira issue to 'Landed' status
git land TEAM-1234

Make a release

  • Support preparation of a new production release
  • Allow for minor bug fixes and preparing meta-data for a release

★ ★ ★

Start a release

To start a release, use the git flow release command. It creates a release branch created from the 'develop' branch.

git start-release

Finish up a release

Finishing a release is one of the big steps in git branching. It performs several actions:

  • Merges the release branch back into 'master'
  • Tags the release with its name
  • Back-merges the release into 'develop'
  • Removes the release branch
git release --finish RELEASE

Don't forget to push your tags with git push origin --tags
