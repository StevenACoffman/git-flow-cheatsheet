Getting Started

Our-Lovely-CLI (olc) needs to be initialized in order to customize your project setup.

★ ★ ★

Initialize

Start using our-lovely-cli by initializing it inside an existing git repository:

git rerun-olc-setup

You'll have to answer a few questions and generate some tokens.

Feature Branches

  • Big Feature or groups of smaller ones for QE
  • Develop new features for upcoming releases
  • Typically longer lived than topic branches

★ ★ ★

Development of new long-lived feature starting from the 'main' branch.

Helps QE to test a group of small changes

  • Create branch 'feature/EPIC-4444'

Make a release

git start-release
  • Support preparation of a new production release
  • Allow for minor bug fixes and preparing meta-data for a release

★ ★ ★

Start a release

To start a release, use the git flow release command. It creates a release branch created from the 'main' branch.

Finish up a release

git release --finish RELEASE

Finishing a release is one of the big steps in git branching. It performs several actions:

  • Merges the release branch back into 'master'
  • Tags the release with its name
  • Back-merges the release into 'main'
  • Removes the release branch

Don't forget to push your tags with git push origin --tags
