Getting Started

Our-Lovely-CLI (olc) needs to be initialized in order to customize your project setup.

★ ★ ★

Initialize

Start using our-lovely-cli by initializing it inside an existing git repository:

git olc-rerun-setup

You'll have to answer a few questions and generate some tokens.

Topic Branch

  • Small Feature or Bugfix

★ ★ ★

Start a new Software Task or Bugfix

Development of a short-lived topic branch from the 'main' branch.

Start developing a new feature with:

git workon TEAM-5432

This action performs the following:

  • Create branch 'TEAM-5432'
  • Switch to the new branch
  • Assign Jira issue to you
  • Transition Jira issue to 'In Progress' status

Make a commit

Make a commit with an interactively formatted commit message
This action performs the following:

  • Select a gitmoji
  • Select a JIRA issue
  • Add a one-line summary
  • Add a multi-line description
  • Add a multi-line test plan
git cm -s -S -a

Start a pull request

Create a pull request for review on GitHub and "start" the Jira issue!
This will transition the Jira issue to "In Review" status.

git pull-request

Land on main

Land a topic branch onto main
This action performs the following:

  • Merges 'TEAM-5432' into main
  • Removes the topic branch
git land

Land a feature

Land a feature branch onto main
This action performs the following:

  • Merges 'feature/EPIC-4444' into main
  • Removes the feature branch
git land --feature

Feature Branches

  • Big Feature or groups of smaller ones for QE
  • Develop new features for upcoming releases
  • Typically longer lived than topic branches

★ ★ ★

Start a new feature

Development of new long-lived feature starting from the 'main' branch.

Allows QE to test one big feature or a group of smaller ones

Start developing a new feature with:

git checkout -b feature/EPIC-4444

This action performs the following:

  • Create branch 'feature/EPIC-4444'

Start working

Begin a Jira Task or Bug fix.

Start developing a new topic branch with:

git workon TEAM-1234

This action performs the following:

  • Create branch 'TEAM-1234'
  • Switch to the new branch
  • Assign Jira issue to you
  • Transition Jira issue to 'In Progress' status

Make a commit

Make a commit with an interactively formatted commit message
This action performs the following:

  • Select a gitmoji
  • Select a JIRA issue
  • Add a one-line summary
  • Add a multi-line description
  • Add a multi-line test plan
git cm -s -S -a

Start a pull request

Create a pull request for review on GitHub and "start" the Jira issue!
This will transition the Jira issue to "In Review" status.

git pull-request

Land on a feature

Land a pull-request on GitHub and transition Jira issue to 'Landed'
This action performs the following:

  • Merges TEAM-1234 into 'feature/EPIC-4444'
  • Removes the topic branch
  • Switches back to 'feature/EPIC-4444' branch
  • Transition Jira issue to 'Landed' status
git land TEAM-1234

Land a feature

Land a feature branch onto main
This action performs the following:

  • Merges 'feature/EPIC-4444' into main
  • Removes the feature branch
git land --feature

Make a release

  • Support preparation of a new production release
  • Allow for minor bug fixes and preparing meta-data for a release

★ ★ ★

Start a release

To start a release, use the git flow release command. It creates a release branch created from the 'main' branch.

git start-release

Finish up a release

Finishing a release is one of the big steps in git branching. It performs several actions:

  • Merges the release branch back into 'master'
  • Tags the release with its name
  • Back-merges the release into 'main'
  • Removes the release branch
git release --finish RELEASE

Don't forget to push your tags with git push origin --tags
